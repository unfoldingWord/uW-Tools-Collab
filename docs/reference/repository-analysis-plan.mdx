## Analysis Framework

### For Each Repository

**Step 1: Repository Metadata**
- Size, language, creation/update dates
- Commit history, branches, tags
- Description and repository statistics

**Step 2: Specification Detection**
- Identify manifest/metadata files
- Parse and classify specification type
- Extract version and format information

**Step 3: File Structure Analysis**
- Complete file inventory by type
- Directory structure mapping
- File naming pattern recognition
- Size distribution analysis

**Step 4: Manifest Deep Dive**
- All manifest fields and their purposes
- Resource Container: dublin_core analysis
- Scripture Burrito: meta and ingredients analysis
- Dependencies and relationships

**Step 5: Content Sampling**
- Sample files from each major type (.usfm, .tsv, .md)
- Content structure analysis
- Format-specific feature detection

**Step 6: Repository Classification**
- Primary format (RC vs Scripture Burrito)
- Repository type (Bible, Notes, Academy, etc.)
- Content scope (Complete Bible, Single Book, etc.)
- Complexity level and target audience
