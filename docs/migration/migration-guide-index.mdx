# Resource Container to Scripture Burrito Migration Guide Index

## Introduction

This index provides a complete reference to all migration guides for converting different types of Resource Container repositories to Scripture Burrito format. Each guide is tailored to the specific characteristics and requirements of different resource types.

## Migration Guide Categories

### Bible Text Migrations
- **[Bible Text RC to SB Migration](bible-text-rc-to-sb-migration.md)**
  - **Covers**: UHB, UGNT, ULT, UST
  - **Complexity**: ⭐ Low
  - **Target Flavor**: textTranslation
  - **Key Features**: USFM preservation, alignment data handling

### Translation Support Migrations
- **[Translation Notes RC to SB Migration](translation-notes-rc-to-sb-migration.md)**
  - **Covers**: TN, OBS-TN
  - **Complexity**: ⭐⭐ Low-Medium
  - **Target Flavor**: parascripturalWordAlignment
  - **Key Features**: TSV preservation, cross-reference enhancement

- **[Translation Words RC to SB Migration](translation-words-rc-to-sb-migration.md)**
  - **Covers**: TW
  - **Complexity**: ⭐⭐ Low-Medium
  - **Target Flavor**: glossary
  - **Key Features**: Nested directory handling, term categorization

- **[Translation Academy RC to SB Migration](translation-academy-rc-to-sb-migration.md)**
  - **Covers**: TA
  - **Complexity**: ⭐⭐⭐ Medium
  - **Target Flavor**: peripheralResource
  - **Key Features**: Three-file article combination, training organization

### Additional Resource Migrations
- **Translation Questions**: Use Translation Notes migration guide (similar TSV structure)
- **Translation Words Links**: Use Translation Notes migration guide (similar TSV structure)
- **Open Bible Stories**: Use Bible Text migration guide with scripturalTextStories flavor

## Migration Planning Workflow

### Step 1: Identify Repository Type

**Repository Analysis**:
- Check `dublin_core.subject` field in manifest.yaml
- Verify `dublin_core.type` field
- Examine file structure and organization
- Refer to quick reference table above

### Step 2: Select Migration Guide

**Guide Selection**:
- Use repository type to select appropriate migration guide
- Note complexity level for planning purposes
- Review key features and considerations

### Step 3: Follow Specific Migration Steps

**Migration Process**:
- Follow step-by-step instructions in selected guide
- Apply repository-specific best practices
- Validate conversion using guide recommendations

### Step 4: Quality Assurance

**Validation Process**:
- Use Scripture Burrito validation tools
- Test with SB-compatible applications
- Verify all content and metadata integrity

- SB metadata generator (JSON)
- File structure analyzer
- Content integrity validator

- **Translation Notes**: TSV parser and cross-reference handler
- **Translation Words**: Directory structure mapper
- **Translation Academy**: Multi-file article combiner

### Validation Tools

**Scripture Burrito Validation**:
- SB schema validator
- Flavor-specific validation
- Content integrity checker
- Relationship validator

## Success Metrics

### Migration Quality Indicators

**Technical Success**:
- All content files properly referenced in ingredients
- Valid SB metadata.json structure
- Correct flavor type assignment
- Preserved cross-references and relationships

**Functional Success**:
- Converted SB loads in SB-compatible tools
- All features work as expected
- Cross-references resolve correctly
- No data loss from original RC

### Long-Term Benefits

**Ecosystem Advantages**:
- Access to growing SB tool ecosystem
- Better interoperability with other Bible tools
- Future-proof format for ongoing development
- Enhanced capabilities over original RC format
