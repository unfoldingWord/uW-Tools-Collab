# Create New Resource Container from First translationCore Book

This guide covers creating a new Resource Container (RC) repository starting with a single translationCore (TC) book. This scenario occurs when no suitable RC exists for the target language/resource combination, requiring a complete RC structure to be built from scratch.

## üéØ **When to Create New RC**

### **Use Cases**
- **No Existing RC**: No RC repository exists for the language/resource combination
- **New Translation Project**: Starting a fresh Bible translation project
- **Organization's First RC**: Creating the first RC repository for an organization
- **Resource Type Mismatch**: Existing RCs don't match the TC resource type

### **Prerequisites**
- translationCore book project completed and validated
- Target language and resource type determined
- Organization/repository hosting location identified
- Understanding of RC structure and requirements

### **Step 1: Analyze TC Source**

**Extract TC Metadata**:
```json

  "project": {"id": "jon", "name": "Jonah"},
  "resource": {"id": "glt", "name": "Gateway Language Translation"},
  "target_language": {
```

**Determine RC Characteristics**:
- **RC Identifier**: `{lang}_{resource}` (e.g., "es-419_glt")
- **Subject**: "Bible" or "Aligned Bible" (if alignment data present)
- **Type**: "bundle" (standard RC type)
- **Format**: "application/zip; type=bundle"

**Repository Naming**:

### **Step 2: Create RC Manifest**

**Generate Dublin Core Structure**:
```yaml
---
dublin_core:
  conformsto: 'rc0.2'
  contributor:
    - 'TC Book Translator'
  creator: 'Organization Name'
  description: 'Bible translation in {Language Name}'
  format: 'application/zip; type=bundle'
  identifier: '{lang}_{resource}'
  issued: '2024-01-15'
  language:
    identifier: '{lang}'
    title: '{Language Name}'
    direction: 'ltr'
  modified: '2024-01-15'
  publisher: 'Organization Name'
  relation:
    - 'en/ult'
    - 'en/ust'
    - 'hbo/uhb?v=2.1.30'
  rights: 'CC BY-SA 4.0'
  source:
    - identifier: '{lang}_{resource}'
      language: '{lang}'
      version: '1.0'
  subject: 'Bible'
  title: '{Language Name} {Resource Name}'
  type: 'bundle'
  version: '1.0'

checking:
  checking_entity:
    - 'Organization Name'
  checking_level: '1'

projects:
  - title: '{Book Name}'
    versification: 'ufw'
    identifier: '{book_id}'
    sort: {book_number}
    path: './{##-BOOK}.usfm'
    categories:
      - 'bible-ot'  # or 'bible-nt'
```

**Customize for TC Source**:
```yaml

  identifier: 'es-419_glt'
```

### **Step 3: Integrate TC Book Content**

**Extract and Convert TC USFM**:
```bash
# Get the working USFM file from TC (contains all translation work)
# translationCore naming: {target_language.id}_{resource.id}_{project.id}_book.usfm
curl -H "Authorization: token {token}" \
  "https://git.door43.org/api/v1/repos/{owner}/{tc-repo}/raw/{branch}/{target_language.id}_{resource.id}_{project.id}_book.usfm"
```

**Create RC Book File**:
```bash
# Determine book number (Jonah = 32)
BOOK_NUM="32"
BOOK_CODE="JON"

# Create RC book file
cp tc_book_content.usfm "${BOOK_NUM}-${BOOK_CODE}.usfm"
```

**Validate USFM Content**:
- Verify `\id` marker matches book
- Check alignment markers preserved
- Validate chapter/verse structure

## ‚ö†Ô∏è **Creation Considerations**

### **Critical Decisions**

**Subject Classification**:
- **"Bible"**: Standard Bible text without alignment
- **"Aligned Bible"**: Bible text with word alignment data
- **Decision**: Based on presence of alignment markers in TC

**Resource Type Mapping**:
- **TC "glt"** ‚Üí RC identifier includes "glt"
- **TC "ust"** ‚Üí RC subject "Bible", resource type preserved
- **TC "ult"** ‚Üí RC subject "Aligned Bible" (typically has alignment)

**Language Information**:
- Use exact language code from TC (`target_language.id`)
- Preserve language name and direction
- Maintain BCP 47 compliance

### **Common Creation Pitfalls**

**Incomplete Manifest**:
- **Issue**: Missing required dublin_core fields
- **Solution**: Use complete template and validate

**Incorrect Book Numbering**:
- **Issue**: Wrong sort order or book number
- **Solution**: Use standard biblical book order reference

**Alignment Data Loss**:
- **Issue**: Stripping alignment markers during transfer
- **Solution**: Preserve TC USFM content exactly

**Dependency Mapping Errors**:
- **Issue**: Incorrect relation field mapping
- **Solution**: Map each TC tsv_relation entry individually

## üìà **Success Criteria**
