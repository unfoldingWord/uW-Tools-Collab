This section provides comprehensive guides for converting translationStudio (tS) repositories to standard Resource Container (RC) format. translationStudio projects focus on single books or story collections with completion tracking and numbered content files.

- **Completion Data Preservation**: Maintain translation progress information

### **translationStudio Characteristics**
Based on the [translationStudio Format Guide](../../4-repository-formats/tool-generated/translationstudio-format-guide):

- **Single Book/Collection Focus**: Bible book or Open Bible Stories
- **Numbered Content Files**: Files named `01`, `02`, `03` (no extensions)
- **Completion Tracking**: `finished_chunks[]` array for progress
- **Two Content Types**: Bible chapters (USFM) or story format
- **Tool-Specific Metadata**: translationStudio-specific manifest format

- Convert numbered files to standard RC naming

- Process `finished_chunks[]` data
- Calculate completion statistics
- Integrate progress information into RC metadata
