## üîÑ **Common Migration Process**

All Bible text migrations follow a similar process:

### **Step 1: Pre-Migration Analysis**
- Verify RC structure and content
- Check for alignment data
- Identify book scope and versification

### **Step 2: Create SB Metadata**
- Convert dublin_core to identification
- Set appropriate flavor type (textTranslation or scripturalTextStories)
- Map language information

### **Step 3: Convert Content**
- Map USFM files to ingredients
- Set mimeType to "text/usfm"
- Create book-level scope objects
- Preserve alignment markers

### **Step 4: Handle Relationships**
- Convert RC relations to SB relationships
- Maintain dependency information
- Preserve version specifications

### **Step 5: Validate**
- Check SB metadata structure
- Verify content integrity
- Test with SB-compatible tools
